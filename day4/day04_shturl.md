\## 二、核心概念



\### 1. 数据持久化



\*\*问题\*\*：之前数据存在内存列表里，程序关闭即丢失。



\*\*方案\*\*：启动时从硬盘文件加载数据，每次修改后立即写入文件。



\*\*流程\*\*：

```

启动 → load\_students() → 读 students.json → 填充内存列表

操作 → 修改内存列表 → save\_students() → 覆盖写 students.json

退出 → 数据已安全存于硬盘

重启 → load\_students() → 恢复全部数据

```



\### 2. 文件读写与上下文管理器



```python

with open(DATA\_FILE, "r", encoding="utf-8") as f:

&#x20;   data = json.load(f)

```



\-  `open()`：打通程序与硬盘文件的通道

\-  `"r"` 读模式 / `"w"` 写模式（覆盖）

\-  `with ... as f:`：上下文管理器，自动关闭文件，防止资源泄漏

\-  `encoding="utf-8"`：确保中文正常读写



\### 3. JSON 序列化与反序列化



| 操作 | 函数 | 方向 |

|------|------|------|

| 反序列化 | `json.load(f)` | 文件 JSON → Python 对象 |

| 序列化 | `json.dump(obj, f)` | Python 对象 → 文件 JSON |



关键参数：

\-  `ensure\_ascii=False`：中文正常显示，不被转义

\-  `indent=2`：格式化缩进，便于人阅读



\### 4. 常量定义



```python

DATA\_FILE = "students.json"

```



\-  全大写命名表示这是一个常量，不应被修改

\-  集中管理：改一处，全局生效，避免漏改错改



\### 5. 异常处理 `try-except`



三种典型异常捕获场景：



| 异常类型 | 场景 | 处理方式 |

|----------|------|----------|

| `json.JSONDecodeError` | JSON 文件格式损坏 | 提示用户，使用空数据集 |

| `IOError` | 文件读写失败（权限/磁盘） | 提示用户，程序不崩溃 |

| `ValueError` | 分数输入字母转整数失败 | 提示重新输入 |



\## 三、今日遇到的Bug与修复



\### Bug 1：函数未调用

\-  \*\*错误\*\*：`load\_students` 只写了函数名，没加括号

\-  \*\*修复\*\*：`load\_students()` 加括号才是调用

\-  \*\*教训\*\*：Python 中函数名不加括号只是引用函数对象，不会执行



\### Bug 2：f-string 中变量写错

\### Bug 3：比较与赋值混淆

\-  \*\*错误\*\*：`ensure\_ascii==False` 双等号是比较运算

\-  \*\*修复\*\*：`ensure\_ascii=False` 单等号才是赋值传参
